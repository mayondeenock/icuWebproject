Question 2: HTML Elements

 1. Five Most Challenging Elements

* `<form>` – It required understanding how inputs work together and how to structure user input properly.
* `<table>` – Managing rows, columns, and headers correctly was a bit complex.
* `<nav>` – Structuring navigation links clearly and logically was challenging.
* `<figure>` and `<figcaption>` – Understanding how to group images with captions properly.
* `<iframe>` – Embedding external content required careful use of attributes.

 2. Use of Semantic Elements

I used semantic elements to organize the website clearly:

* `<header>` for the top section with the title and navigation
* `<nav>` for menu links
* `<section>` to divide content into logical parts
* `<article>` for individual pieces like projects
* `<footer>` for contact info and copyright

This made the website easier to read and understand.

 3. Most Useful Element

The `<section>` element was the most useful because it helped divide the website into clear, meaningful parts, improving both readability and structure.



 Question 3: HTML Attributes

 1. Three Essential Attributes

* `href` – Used in links to navigate between pages
* `src` – Used to display images and media
* `alt` – Provided descriptions for images (important for accessibility)

 2. Class vs ID

* `class` was used for grouping multiple elements with similar styles or purposes
* `id` was used for unique elements (like navigation anchors or specific sections)

 3. Most Important Attribute for User Experience

The `alt` attribute improved user experience the most because it helps visually impaired users understand images and also displays text if images fail to load.
